Pipeline

How it works

The translator operates directly on PDF internals without OCR or rasterization. Mathematical notation passes through untouched.

1

Extract

Parse every text span in the PDF. Classify each as translatable text (SFRM, SFBX, SFTI fonts) or math notation (CMMI, CMSY, CMEX, etc.) to be preserved exactly.

2

Translate

Send text to Google Translate with opaque XXXM0XXX markers standing in for math spans. Google preserves these tokens, and the translator restores them after translation.

3

Render

Redact original text from the PDF, then re-render using CMU Serif for text and Latin Modern Math for symbols, all as real vector glyphs with proper Unicode math code points.

Details

Features

∫

Vector math symbols

All math notation rendered as real vector text using Latin Modern Math, without rasterized images.

ξ

Proper math italics

Variables use Unicode Mathematical Italic code points for correct typographic rendering.

¶

Paragraph-level translation

Pure-text blocks are merged into paragraphs before translation for more natural output.

···

TOC preservation

Table of contents dot leaders, page numbers, and hyperlink annotations are regenerated correctly.

B

Style preservation

Bold, italic, and bold-italic text styles are detected and preserved through translation.

↔

Any source language

Translate to English from any language supported by Google Translate.

Installation

Command line

Clone the repo and run with uv. No configuration needed.

# Clone and install
$ git clone https://github.com/joshuaswanson/latex-pdf-translator
$ cd latex-pdf-translator

# Translate a PDF (default: French to English)
$ uv run main.py lecture-notes.pdf

# Translate German to Spanish
$ uv run main.py notes.pdf --source de --target es
